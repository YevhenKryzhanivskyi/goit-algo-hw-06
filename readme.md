Порівняння алгоритмів DFS (Depth-First Search) та BFS (Breadth-First Search) для пошуку шляхів у графі метро Лондона.

Опис графа

Граф побудовано на основі фрагмента мережі метро Лондона. Вершини — станції, ребра — з’єднання між сусідніми станціями на лініях метро. Додатково позначено пересадкові вузли.

Алгоритми

DFS (пошук у глибину): рухається максимально глибоко по одному напрямку, перш ніж повернутися назад.

BFS (пошук у ширину): розширює пошук рівень за рівнем, гарантуючи знаходження найкоротшого шляху (за кількістю ребер).

Результати пошуку

Приклад: шлях від Notting Hill Gate до London Bridge

DFS шлях:

Notting Hill Gate -> Queensway -> Lancaster Gate -> Marble Arch -> Bond Street -> Green Park -> Westminster -> Waterloo -> Southwark -> London Bridge

BFS шлях:

Notting Hill Gate -> Bond Street -> Green Park -> Westminster -> Waterloo -> Southwark -> London Bridge

Пояснення різниці

DFS обирає перший можливий напрямок і йде вглиб, тому шлях може бути довший і не оптимальний.

BFS розглядає всі варіанти на кожному рівні, тому знаходить найкоротший шлях у незваженому графі.

У прикладі:

DFS пройшов через більшу кількість станцій на Central line, перш ніж дістатися до пересадки.

BFS одразу знайшов коротший маршрут через пересадку на Bond Street → Jubilee line.

Висновки

DFS корисний для обходу графа та знаходження будь-якого можливого шляху, але не гарантує мінімальності.

BFS завжди знаходить найкоротший шлях у незважених графах, що особливо важливо для транспортних мереж.

Для задач навігації у метро чи транспортних системах BFS є більш практичним, адже мінімізує кількість пересадок і кроків.

Таким чином, BFS забезпечує оптимальний маршрут, тоді як DFS може дати довший, але теж коректний шлях.
